# Syntra Development Guidelines

## Coding Standards

- Adhere to Swift API Design Guidelines
- Use meaningful names and detailed documentation
- Maintain consistent formatting with SwiftFormat
- Write unit tests for all new functionality

## Architecture Principles

- Separation of concerns across packages
- Dependency injection for testability
- Error handling with typed errors
- Performance considerations for LLM workflows

## Contribution

1. Fork and branch from main
2. Implement changes with tests
3. Update documentation
4. Submit pull request
