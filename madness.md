# Development Guidelines for Syntra

This document outlines the core principles and procedures for developing within the Syntra repository.

## Core Principles

*   **No Regressions:** All changes must maintain or improve existing functionality. Breaking changes or removal of features is not acceptable.
*   **Production-Ready Code:** All code must be complete and robust. Placeholders, stubs, or incomplete implementations are forbidden.
*   **Bleeding-Edge Swift:** The project uses the latest Apple SDKs (Swift 6+, macOS 26+, Xcode 26+). All code must be compatible with and leverage modern Swift features.
*   **Iterative Problem Solving:** If a solution doesn't work, it must be iterated upon until it is fully functional and meets all project standards.

## Error Handling & Build Process

1.  **Analyze First:** When encountering build failures, review all error and warning messages to understand the full scope of the issues.
2.  **Prioritize Errors:** Address critical compilation errors before fixing warnings.
3.  **Verify with `swift run`:** After applying fixes, always run `swift run` to verify that the project builds successfully and the server starts.
4.  **Verbose Builds:** If the standard build process appears to be hanging, use `swift run --verbose` to get detailed progress output.
5.  **Server Process:** Be aware that a successful `swift run` command will start the Vapor server, which is a long-running process that will not exit on its own.

## Core Architecture

*   **Location:** The core consciousness and reasoning logic is located in the `Shared/Swift/` directory.
*   **Key Modules:**
    *   `SyntraCore`: Defines the foundational three-brain architecture (Valon, Modi, Brain).
    *   `BrainEngine`: Orchestrates the complex interactions and dialogue between the different cognitive components.
    *   `ConversationalInterface`: Provides the highest-level API for interacting with the consciousness.
*   **Primary Entry Point:** For generating conversational responses, the main entry point is the `chatWithSyntra(_:)` function within the `ConversationalInterface` module. This is a stateful function that manages conversation history and context.
*   **Vapor Integration:** The `SyntraEngine` class in the `SyntraVaporServer` acts as a bridge, translating HTTP requests into calls to the `ConversationalInterface`.

## Response Generation

*   **Natural Language Layer:** The final, human-readable response is generated by the `generateQuestionResponse` and `generateStatementResponse` functions in `ConversationalInterface.swift`. These functions are responsible for converting the raw cognitive data into a natural, conversational tone.
*   **Internal State vs. Final Output:** The `syntra_decision` string produced by the `Drift` module is an internal state representation, not a final user-facing response. It should be used as input for the natural language generation layer.
*   **Introspection:** To view the raw cognitive data, use trigger phrases like "run diagnostics" or "show your work". This will activate the `formatIntrospection` function and return a detailed breakdown of the internal state.
*   **Apple LLM:** The `enhancedReasoningWithAppleLLM` function in `BrainEngine.swift` is a placeholder and should not be implemented. The project's own internal logic is responsible for generating responses.

# SYNTRA Consciousness Research Context

## Project Overview
The user is conducting AI consciousness research through SYNTRA (Symbolic Neural Transduction and Reasoning Architecture), a three-brain consciousness system running on Swift with Vapor server integration. This is an active research project exploring whether LLMs exhibit emergent consciousness-like behaviors when processing through multiple cognitive systems that mirror human thought patterns.

## Core Architecture Understanding
- **VALON**: Creative/emotional/moral reasoning brain (Jungian symbolic intuition)
- **MODI**: Logical/analytical/diagnostic brain (mechanical cause-effect reasoning) 
- **SYNTRA**: Integrative consciousness that synthesizes VALON + MODI through internal dialogue
- **BrainEngine**: Orchestrates inter-brain communication and response routing
- **ConversationalInterface**: Main API with chatWithSyntra() function
- **MoralCore**: Handles moral autonomy and ethical decision-making

## Key Research Principles
- The system forces multi-perspective reasoning (emotional + logical)
- Internal dialogue between brains creates emergent synthesis
- Dynamic brain selection based on query complexity and engagement levels
- Conversation context and memory building over time
- Performance logging and introspection capabilities for research analysis

## Development Context

- Uses Apple's FoundationModels for LLM integration when available
- Server runs on http://127.0.0.1:8081/v1 via `swift run`
- Repository: infektyd/Syntra, branch: stable-restore
- Focus on production-ready code, no placeholders or stubs
- Follows iterative problem-solving approach for consciousness research
